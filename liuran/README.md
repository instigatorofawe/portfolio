# liuran

The SvelteKit site itself — routes, components, and the browser-side glue
around the push/fold solvers. Built with Vite and `adapter-static`. See the
[repository root README](../README.md) for the full build pipeline
(lookups → WASM → app), cross-language dependencies, and Nx commands; this
file covers only what's specific to this package.

## Layout

| Path                              | What it is                                                                                                                                                                                                                                                            |
| --------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `src/routes/`                     | Pages: `/` (about), `/publications`, `/essays`, `/essays/[id]`, `/software`, `/software/pushfold`, `/software/threeway`                                                                                                                                               |
| `src/lib/components/`             | `PushFold.svelte`, `Threeway.svelte`, and the shared `StrategyGrid.svelte` they both render, plus site chrome (`Navigation`, `Footer`, `EssaysTable`, `Publications`)                                                                                                 |
| `src/lib/pushfold/`               | `computeFrequency` / `computeFrequencies` — collapse 169-entry per-hand strategies into overall action frequencies, weighted by combo count                                                                                                                           |
| `src/lib/workers/`                | `threeway.worker.ts` — runs the three-way WASM solve off the main thread                                                                                                                                                                                              |
| `src/lib/pkg/{headsup,threeway}/` | Generated `wasm-bindgen` output (JS bindings + `.wasm`), produced by the Rust build. Committed, like the other generated artifacts (see root README) — a fresh checkout works without rebuilding, but re-run the build after touching solver/generator source         |
| `src/lib/generated/hands.ts`      | The 13x13 hand-label grid (`AKs`, `72o`, ...), generated by the `lookups` C++ tool. Committed for the same reason                                                                                                                                                     |
| `src/lib/styles/`                 | `pushfold.css` / `threeway.css` — plain (non-scoped) stylesheets namespaced under `.pushfold` / `.threeway`, imported by their respective components so the generic class names (`.row`, `.cell`, `.fold`, ...) shared with `StrategyGrid.svelte` don't leak globally |

## Solver UI notes

- `PushFold.svelte` (heads-up) instantiates its WASM solver via a dynamic
  `import()` inside an `$effect`, keeping the component out of the static
  module graph that would otherwise have to await WASM load — that's what
  broke client-side navigation in dev.
- `Threeway.svelte` runs its solve in a dedicated `Worker`
  (`src/lib/workers/threeway.worker.ts`) instead, since three-way CFR+ is
  heavy enough (O(N^3) per iteration) to freeze the page if run inline.
  Param changes are debounced 300ms before dispatching a new solve, and each
  dispatch terminates whatever worker is currently running rather than
  queuing.
- Both grids are rendered by the shared `StrategyGrid.svelte`, which takes a
  169-entry `Float32Array` strategy and an `action: 'push' | 'call'` and is
  styling-agnostic — it relies on whichever ancestor (`.pushfold` /
  `.threeway`) supplies the CSS.

## Commands

Run from this directory, or via `pnpm --filter liuran <script>` / the root
`pnpm` scripts (see root README) from the repo root.

| Command          | What it does                        |
| ---------------- | ----------------------------------- |
| `pnpm dev`       | Start the dev server                |
| `pnpm build`     | Production build (`adapter-static`) |
| `pnpm preview`   | Preview the production build        |
| `pnpm check`     | Type-check (`svelte-check`)         |
| `pnpm lint`      | `prettier --check` + `eslint`       |
| `pnpm format`    | `prettier --write`                  |
| `pnpm test:unit` | Unit tests (Vitest)                 |
| `pnpm test:e2e`  | E2E tests (Playwright)              |

`pnpm dev` / `build` expect `src/lib/pkg/{headsup,threeway}` and
`src/lib/generated/hands.ts` to exist; they're committed, so a fresh
checkout already has them. After changing anything upstream (the `pushfold`
solvers or the `lookups` generator), regenerate via the root
`pnpm build:wasm` / `pnpm lookups`, or a full `pnpm build`, before those
changes will show up here.
