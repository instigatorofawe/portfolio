# Ran Liu - Portfolio Site

A SvelteKit portfolio site with in-browser push/fold poker solvers (heads-up
and three-way) compiled to WebAssembly. The solvers' lookup tables are
generated by a C++ program.

All commands below are run from the repository root unless noted otherwise.

## Layout

| Path       | What it is                                                              |
| ---------- | ---------------------------------------------------------------------- |
| [`liuran/`](liuran/README.md)  | SvelteKit app (the website), built with Vite and `adapter-static`     |
| [`pushfold/`](pushfold/README.md)| Rust workspace for the push-fold solvers: `shared` (constants + generated tables), `headsup`, and `threeway`, each compiled to WASM |
| [`lookups/`](lookups/README.md) | C++ generator that emits the solvers' lookup tables as Rust (or TypeScript) source |
| `scripts/` | Helper scripts (`build-headsup.sh`, `build-threeway.sh`)             |

## Dependencies

| Tool                       | Used for                          | Install                                                        |
| -------------------------- | --------------------------------- | ------------------------------------------------------------- |
| Node.js (>= 20)            | Running the app / Nx              | https://nodejs.org or `nvm install`                           |
| pnpm (>= 10)               | JS package manager / scripts      | `corepack enable` or `npm i -g pnpm`                          |
| Rust + Cargo (edition 2024, >= 1.85) | Building the solver     | https://rustup.rs                                            |
| `wasm32-unknown-unknown`   | WASM compile target               | `rustup target add wasm32-unknown-unknown`                   |
| `wasm-bindgen-cli`         | Generating JS bindings for the WASM | `cargo install wasm-bindgen-cli`                            |
| CMake (>= 3.21)            | Building the C++ lookup generator | `brew install cmake`                                          |
| A C++23 compiler           | Compiling the generator           | Xcode CLT (`xcode-select --install`) / recent clang or gcc   |
| Git                        | CMake fetches OMPEval at configure time | (preinstalled on most systems)                         |

Optional / dev-only:

| Tool         | Used for                                              | Install                       |
| ------------ | ---------------------------------------------------- | ----------------------------- |
| libomp       | Parallelizing the generator (`EquityGenerator::Solve`); without it it runs serially | `brew install libomp` (macOS) |
| pre-commit   | Running the git pre-commit hooks                    | `brew install pre-commit`     |
| clang-format | C++ formatting hook (uses Xcode's via `xcrun` if present) | bundled with Xcode CLT / LLVM |

Nx itself is a project dev-dependency, so it's installed by `pnpm install` at the
repository root — no global install needed.

## Setup

```
pnpm install   # installs every workspace package (Nx, the SvelteKit app, ...)
```

Then add the Rust target and `wasm-bindgen` if you don't have them (see the table
above), and install `pre-commit` hooks if you intend to commit:

```
pre-commit install
```

## Build pipeline

`pnpm build` (→ `liuran:build`) has Nx orchestrate the following as a dependency
graph, each stage caching independently:

1. **`lookups:generate-hands`** (depends on `lookups:generate-lookups`) —
   configures & builds the C++ generator, runs it to emit the heads-up equity
   and matchup tables into `pushfold/shared/src/generated/headsup/`, then a
   second pass emits the hand-label grid to `liuran/src/lib/generated/hands.ts`.
2. **`pushfold:build-headsup-wasm`** — compiles the heads-up Rust solver to WASM
   and runs `wasm-bindgen`, emitting JS bindings into `liuran/src/lib/pkg/headsup/`.
3. **`liuran:build`** — builds the SvelteKit app.

The three-way lookup generation (`lookups:generate-threeway-lookups`) is
**deliberately not wired into this graph**: enumerating the ~819K three-way
matchups takes upwards of 40 minutes even on fast hardware, so making any
build target depend on it would mean CI (and any cold-cache local build)
pays that cost on every run. `pushfold:build-threeway-wasm` isn't part of
the top-level graph either, for the same reason — it depends transitively
on the same generator target.

This mostly doesn't matter day to day: every generated artifact in this
pipeline (the C++ generator's Rust/TS output, and the WASM bindings built
from it) is committed rather than gitignored, so a fresh checkout works
without running any of it. Regenerate only after changing the solver or
generator source that a given artifact is derived from — three-way
regeneration is always an explicit, manual `pnpm lookups:threeway`
(see [`pushfold/README.md`](pushfold/README.md) and
[`lookups/README.md`](lookups/README.md)).

## Commands

| Command                  | What it does                                                       |
| ------------------------ | ------------------------------------------------------------------- |
| `pnpm dev`               | Start the SvelteKit dev server                                    |
| `pnpm build`             | Orchestrated build (hands + heads-up WASM → app) via Nx; see [Build pipeline](#build-pipeline) |
| `pnpm build:web`         | Build just the app, skipping the Nx pipeline                      |
| `pnpm build:wasm`        | Build just the heads-up WASM solver (runs lookup generation first) |
| `pnpm build:wasm:threeway` | Build the three-way WASM solver (not part of `pnpm build`; see [Build pipeline](#build-pipeline)) |
| `pnpm lookups`           | Generate the heads-up Rust lookup tables                          |
| `pnpm lookups:threeway`  | Generate the three-way Rust lookup tables — **40+ minutes**, run by hand only |
| `pnpm preview`           | Preview the production build                                      |
| `pnpm check`             | Type-check the app (`svelte-check`)                                |
| `pnpm lint`              | Lint + format-check the app                                       |
| `pnpm format`            | Format the app with Prettier                                      |
| `pnpm test`              | Run the app's unit + e2e tests                                    |
| `pnpm test:unit`         | Run unit tests (Vitest)                                           |
| `pnpm test:e2e`          | Run e2e tests (Playwright)                                        |
| `pnpm test:cpp`          | Build & run the `lookups` GoogleTest suite                        |
| `pnpm test:rust`         | Run the `pushfold` workspace's Rust tests                         |
| `pnpm ci:local`          | Everything CI runs: build, check, lint, `test:cpp`, `test:rust`, app unit + e2e tests |
| `pnpm clean:generated`   | Delete generated Rust/TS/WASM output (`pushfold/shared/src/generated/{headsup,threeway}`, `liuran/src/lib/generated`, `liuran/src/lib/pkg`) |
| `pnpm clean`             | `clean:generated`, plus reset the Nx cache and delete cmake/cargo/SvelteKit build dirs |
| `pnpm rebuild`           | `clean` then a full `build` from scratch                          |

### Forcing a rebuild

Nx caching and the cmake/cargo incremental builds are separate layers:

| Command                        | Effect                                                              |
| ------------------------------ | ------------------------------------------------------------------ |
| `pnpm build --skip-nx-cache`   | Re-run all stages, but cmake/cargo stay incremental (fastest)      |
| `npx nx reset`                 | Wipe Nx's cache store and daemon                                   |
| `pnpm rebuild`                 | Clear the Nx cache **and** the cmake/cargo build dirs, recompiling everything |
